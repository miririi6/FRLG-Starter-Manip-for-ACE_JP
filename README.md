# 概要
FRLG任意コード実行RTAにおいて、バグポケモンの種となりつつ高個体値な御三家を検索するためのソフトウェアです。  
個体値、性格、必要経験値等でフィルタをかけてバグ素材となるポケモンを検索できます。  
バグポケモンの作成方法には経験値をつかうものと、努力値を使うものがありますが、現状このソフトでは経験値にのみ対応しています。  
バグポケモン用のフィルターを外して、通常の高個体値を検索するモードも搭載しています。
# 使い方
基本的な使い方は[Gen 3 Starter Predictor](https://www.speedrun.com/pkmnfrlg/resources/2ssx4/)と同じです。  
## ACE表(右側のデータ)について
素材ポケモンの経験値を表示されている数値に合わせて、メールバグで3個目(右上)にワード3、5個目(真ん中)にワード5の文字を入れることでACEバグポケモンが生成されます。  
必要経験値が少ない順に表示されます。  
最小経験値+87未満かつ偶数の数値はフィルタで弾いているため、表示されません。[^1]
[^1]: 育て屋で調整できる最小の奇数が87。
### 注意事項
メールワードはwords.csvとpoke_words.csvに保存されている値を使用します。  
poke_wordsには、一部確定では登録されないものが含まれています。  
例えばゼニガメを選んだ場合は、ヒトカゲ・リザードは使用できません。  
その他、パラスやドードーなどは野生エンカウントしなかった場合は使用できません。
## 画像認識(v1.1にて追加)
[OpenCvSharp](https://github.com/shimat/opencvsharp)を使用しています。  
ウィンドウを選択肢、相対位置を一定間隔でキャプチャして数値を確認します。  
ウィンドウ右上からの相対位置でキャプチャしているので、サイズ変更等をすると認識できなくなります。  
また、上に他のウィンドウが重なっていると認識できません。
### 範囲設定方法
設定画面で画像認識するウィンドウを選択し、範囲選択ウィンドウを起動します。  
範囲選択ウィンドウは、右下をドラッグでサイズ変更、それ以外をドラッグで移動できます。
範囲選択ウィンドウをゲーム画面の数字部分に合わせ、範囲決定ボタンをクリックします。
設定は自動保存されませんので、メインウィンドウから設定ファイルを保存してください。
### その他設定
#### 実行間隔
キャプチャ間隔を指定します。0.1秒から2秒まで設定可能です。
#### 閾値
画像の一致判定に使う閾値を指定します。0.5から1.0まで設定可能です。
### 参照画像について
NumberImages内のpngファイルを参照しています。ファイル名を直指定しているので、png以外の拡張子は参照できません。  
環境によりうまく取れない場合もあるので、うまく読み取れない場合は各自の環境で画像を用意して下さい。
### 仕様
キャプチャ画像内に各数字画像と一致する部分が無いか、0から順に確認します。[^2]  
既に一致判定が出た場合、その部分は黒塗りにして、同じ数字から再度確認します。  
数値が5個判定された場合、自動でトレーナーID欄に張りつけて検索します。
[^2]: 0から順に確認する仕様上、閾値が低いと低い数値として判定される可能性が高くなります。（例：8が3と判定される）
## FlowTimerへの自動数値入力(v1.2にて追加)
計算完了時とクリック時、自動的にFlowTimerに数値を入力してFlowTimerをアクティブ状態にします。  
ウィンドウタイトルの完全一致検索後、ウィンドウ内の座標から入力欄を探すという少々強引な実装なので、環境によっては動かない可能性があります。  
設定画面からON/OFFを選択できます。
## 設定について
設定タブから設定変更画面を開くことができます。  
設定変更時は各リストが再計算されます。
### ポケモン選択
Lv5,Lv6ステータスの表示に使用します。  
### 検索モード
バグ無しモードにすると、バグポケモン関連の設定やフィルタが無効化されます。  
ACE表も非表示になり、ウィンドウが小さくなります。
### バージョン
バージョンによりバグポケモンのIDが一部変わります。
### バグ素材ポケモン選択
ワードごとの必要経験値が変わります。[^3]
[^3]: トレーナーID、性格値、ワード3の組み合わせでバグポケモンが決まるため、素材ポケモンの種族は経験値調整にのみ影響する。
## 設定ファイルについて
個体値や性格を含め、各種設定値は設定ファイル「config.json」に保存されています。  
設定変更後、config.jsonを上書き保存することで次回起動時の設定を更新できます。
